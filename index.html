Setup Instructions

Configure Hermes Agent, Hermes Proxy, and OpenCode to use this optional router/dashboard layer.

Recommended First Run
Start with Hermes Proxy or OpenRouter, then add direct frontier providers later.
ModelFoundry works best when you get one strong provider lane working first, keep your primary Hermes/OpenCode stack intact, and treat the router as an optional OpenAI-compatible lane.

Hermes Agent

Automatic

  1. Run model-foundry onboard (or modelrelay onboard for legacy binary compatibility).
  2. Accept the Configure local Hermes Proxy endpoint? prompt if you want ModelFoundry to route through Hermes-managed OAuth subscriptions.
  3. Start the router with model-foundry, then configure Hermes Agent with the commands below.

Manual

Use Hermes' own config CLI so profile files stay in the canonical Hermes format. Reference: https://hermes-agent.nousresearch.com/docs/user-guide/configuration.
# Point Hermes Agent at ModelFoundry's OpenAI-compatible router
hermes config set model.provider custom
hermes config set model.base_url http://127.0.0.1:7352/v1
hermes config set model.api_key no-key
hermes config set model.default auto-fastest
hermes chat -q "Say hi through ModelFoundry" -Q

Hermes Proxy upstream

OAuth passthrough

  1. Start Hermes Proxy separately: hermes proxy start --provider auto --host 127.0.0.1 --port 8648.
  2. In Settings, click Add Hermes Proxy under OpenAI-Compatible endpoints.
  3. Leave the model blank for discovery, unless you intentionally want to pin one model from /v1/models.

Docker env

MODELFOUNDRY_HERMES_PROXY_BASE_URL=http://host.docker.internal:8648/v1
MODELFOUNDRY_HERMES_PROXY_API_KEY=unused
MODELFOUNDRY_HERMES_PROXY_MODEL=

OpenCode (Opencode)

Automatic

  1. Run model-foundry onboard (or modelrelay onboard for legacy compatibility).
  2. When asked Auto-configure OpenCode now? [Y/n], answer Y.
  3. Start the router with model-foundry (or modelrelay for legacy compatibility).

Manual

Put this in ~/.config/opencode/opencode.json: 
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "router": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "modelfoundry",
      "options": {
        "baseURL": "http://127.0.0.1:7352/v1",
        "apiKey": "dummy-key"
      },
      "models": {
        "auto-fastest": {
          "name": "Auto Fastest"
        }
      }
    }
  },
  "model": "router/auto-fastest"
}
